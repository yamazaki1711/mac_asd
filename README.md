# MAC ASD v12.0 — Автоматизированный комплекс восстановления исполнительной документации

**MAC ASD** (Multi-Agent Construction AI System) — полностью локальная ИИ-система, которая заходит на проблемный объект капитального строительства и **выводит его к полному комплекту исполнительной документации**, готовому к сдаче заказчику.

## Проблема

На миллиардных объектах ИД — в хаосе. Оплата заблокирована. Генподрядчик выставляет претензии. Сроки горят. Классическое решение — выездная команда 15–20 человек на месяцы. Результат — неполный комплект, ошибки, несогласованности между документами.

## Решение

**4 оператора + Mac Studio + сканер.** Срок: 2–3 недели. Стоимость услуги: 25–50 млн ₽ при стоимости объекта от 1 млрд ₽.

АСД не генерирует отдельные документы. АСД **восстанавливает весь комплект** — 13 типов документов по Приказу 344/пр, связанных хронологически, логически и нормативно в единый непротиворечивый пакет.

> **Человек принимает финальное решение.** Система — штурман, а не автопилот.

## Рабочий цикл

```
┌──────────┐    ┌───────────┐    ┌────────────┐    ┌──────────┐    ┌───────┐
│  FEEDING │───→│ INVENTORY │───→│ RESTORATION│───→│ APPROVAL │───→│ PRINT │
│ Сканиро- │    │ Инвента-  │    │ Генерация  │    │ Строй-   │    │ Печать│
│ вание    │    │ ризация   │    │ недостаю-  │    │ контроль │    │       │
│ всех     │    │ что есть /│    │ щих ИД     │    │ человек  │    │       │
│ бумаг    │    │ чего нет  │    │            │    │ в цикле  │    │       │
└──────────┘    └───────────┘    └────────────┘    └──────────┘    └───────┘
```

**Feeding** — сканируются ВСЕ бумаги на объекте: чертежи, сертификаты, акты, журналы, накладные. Хаос — это нормально.

**Inventory** — система классифицирует каждый документ (18 типов), строит граф связей (АОСР → сертификат → партия → ТТН) и показывает **дельту**: чего нет, что лишнее, где разрывы в цепочках.

**Restoration** — генерация недостающего. Два режима:
- **Forensic-восстановление** — из хаоса существующих документов: обратная реконструкция журналов из фрагментов, заполнение разрывов в шлейфах АОСР
- **Генерация с нуля** — когда документов не было никогда: полный комплект по ТТК и ПОС

**Approval** — человек в цикле. Жизненный цикл документа: DRAFT → REVIEW → APPROVED → PRINTED. Система не подписывает за людей.

**Print** — готовый комплект на бумаге, согласованный и непротиворечивый.

## Два стратегических режима

| | Восстановление ИД | Сопровождение строительства |
|---|---|---|
| **Сценарий** | Проблемный ОКС, ИД в хаосе, оплата заблокирована | Текущий объект, ИД формируется параллельно стройке |
| **Ценность** | 25–50 млн ₽ за восстановление полного комплекта | Снижение трудозатрат на ИД в 3–5 раз |
| **Команда** | 4 оператора + Mac Studio | ПТО + Mac Studio |
| **Срок** | 2–3 недели | Постоянно |

## Архитектура системы

### 7+1 мультиагентный экипаж

| Агент | Что делает | Модель |
|-------|-----------|--------|
| **Hermes** (PM) | Оркестрация, вердикты, маршрутизация задач | Llama 3.3 70B |
| **ПТО** | Чертежи, ВОР, спецификации, OCR, 20 видов работ | Gemma 4 31B (VLM) |
| **Юрист** | БЛС (58 ловушек), контракты, претензии | Gemma 4 31B (shared) |
| **Сметчик** | ФЕР/ТЕР, НМЦК, рентабельность | Gemma 4 31B (shared) |
| **Закупщик** | Тендеры, поставщики | Gemma 4 31B (shared) |
| **Логист** | Снабжение, КП, доставка | Gemma 4 31B (shared) |
| **Делопроизводитель** | Регистрация, архив, шаблоны ИД | Gemma 4 E4B |
| **Auditor** (RedTeam) | Forensic-проверки, стройконтроль | Llama 3.3 70B |

> Gemma 4 31B — одна копия в памяти (~23 GB) на 5 агентов. 3 уникальные модели, ~66 GB RAM.

### Три ключевых принципа

**1. Документы следуют в технологической последовательности.** Журнал → АОСР → КС-2. При утрате исходных система восстанавливает их из первичных свидетельств на объекте, а не из производных документов.

**2. АОСР генерируются цепочками по ТТК**, а не поштучно. Каждый акт связан с предыдущим и следующим этапом работ.

**3. Человек в цикле.** DRAFT → REVIEW → APPROVED → PRINTED. Система не подписывает за людей.

### Ключевые модули

| Модуль | Назначение |
|--------|------------|
| **Ingestion Pipeline** | Сканы → OCR → Классификация (18 типов) → Извлечение сущностей → Граф связей |
| **Forensic KAG** | Граф связей между документами (АОСР ↔ сертификаты ↔ партии ↔ ТТН ↔ накладные). Автоматический поиск несоответствий, разрывов цепочек, нарушений нормативных требований |
| **IS Generator** | Исполнительные схемы: DXF-парсинг → отклонения → ГОСТ-штамп → SVG/PDF (16 модулей, 99+ тестов) |
| **PPR Generator** | Проект производства работ: ПЗ по уровням зависимостей + 6 ТТК-генераторов + графика |
| **Output Pipeline** | DOCX-генерация по 344/пр: АОСР, нумерация, Times New Roman 12pt |
| **HermesRouter** | 3-стадийная модель решений: weighted scoring → LLM reasoning → veto |
| **БЛС** | 58 ловушек в договорах строительного подряда (10 категорий) + pgvector RAG |
| **Lessons Learned** | Институциональная память: БД → RAG-инъекция → Skill Mutation (3 уровня) |
| **WorkTypeRegistry** | 20 видов работ → ФЕР/ТЕР маппинги, состав ИД по каждому виду |

### Forensic-анализ

Система проверяет комплект ИД на непротиворечивость и соответствие нормативным требованиям. Области проверок:

- **Целостность шлейфа документов** — АОСР → сертификаты → партии → ТТН → накладные: разрывы, отсутствующие звенья, дублирование
- **Хронологическая непротиворечивость** — даты работ, датировка сертификатов, последовательность актов по технологической карте
- **Количественные соответствия** — объёмы по АОСР vs ВОР vs сертификаты vs КС-2: превышения, недостачи, ксерокопирование
- **Нормативное соответствие** — ГОСТ на материалы, актуальность марок, снятые с производства изделия, заменённые нормативные документы
- **Перекрёстные связи** — один и тот же сертификат в разных АОСР, отсутствие входного контроля, пропуски в журналах при наличии актов

Каждое нарушение получает приоритет (CRITICAL / HIGH / MEDIUM / LOW) и формирует задачу оператору через Guidance System. Набор проверок расширяем — система учится на каждом объекте через Lessons Learned.

## Быстрый старт

```bash
# 1. Инфраструктура
cd infrastructure && docker compose up -d    # PostgreSQL 16 + pgvector

# 2. Окружение
cp .env.example .env
export ASD_PROFILE=dev_linux    # или mac_studio

# 3. Прогнать E2E forensic-тест
PYTHONPATH=. python tests/test_e2e_forensic.py

# 4. Все тесты
.venv/bin/python -m pytest tests/ -v

# 5. MCP-сервер
python -m mcp_servers.asd_core.server
```
