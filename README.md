# MAC ASD v11.0 — Мультиагентная ИИ-система строительного субподряда

**MAC ASD** (Multi-Agent Construction AI System for Automated Subcontracting Decisions) — полностью локальная, оффлайн-ориентированная система автоматизации работы департамента строительства. Спроектирована для запуска на **Mac Studio M4 Max 128GB** с использованием нативного фреймворка **MLX** для инференса LLM.

> Не заменяет специалистов. **Усиливает** их. Человек всегда принимает финальное решение.

---

## Архитектура: 7 специализированных агентов

Система построена на базе **LangGraph StateGraph** — каждый агент является узлом графа, а оркестратор **Hermes** маршрутизирует задачи между ними. Все LLM-вызовы проходят через единый **LLMEngine** (MLX).

| Агент | Роль | Модель (Mac Studio) | Ключевая специализация |
|-------|------|---------------------|----------------------|
| **Hermes** | Оркестратор / PM | Llama 3.3 70B 4-bit (MLX) | Маршрутизация, вердикты, эскалация |
| **ПТО** | Инженер ПТО | Qwen2.5-VL-72B 4-bit (MLX) | ВОР, чертежи, спецификации, Vision |
| **Сметчик** | Сметный расчет | Qwen3 32B 4-bit (MLX) | ФЕР/ТЕР, НМЦК, рентабельность |
| **Юрист** | Юридическая экспертиза | Qwen3 32B 4-bit (MLX) | БЛС, контракты, риски, претензии |
| **Закупщик** | Тендеры и закупки | Qwen3 32B 4-bit (MLX) | Поиск лотов, НМЦК, стратегия торгов |
| **Логист** | Снабжение | Qwen3 32B 4-bit (MLX) | Поставщики, цены, КП, поставки |
| **Делопроизводитель** | Документооборот | Gemma 4 9B 4-bit (MLX) | Регистрация, классификация, ИД |

> Qwen3 32B — общая модель в памяти для Сметчика, Юриста, Закупщика и Логиста (Shared Memory, MLX).

## Два режима работы

### 1. Поиск подходящих лотов
Мониторинг тендерных площадок → оценка НМЦК → извлечение ВОР → поиск поставщиков → сметный расчет → юридическая экспертиза → вердикт о участии.

### 2. Сопровождение строительства
Вводный контроль → оперативный контроль объемов → финансовый контроль (КС-2/КС-3) → логистическое обеспечение → претензионная работа → завершение объекта.

## Технологический стек

| Слой | Технология | Назначение |
|------|-----------|------------|
| **Оркестрация** | LangGraph StateGraph | Управление потоками между агентами |
| **LLM Engine** | LLMEngine (MLX) | Унифицированный интерфейс |
| **LLM-модели** | Llama 70B, Gemma 4 31B/9B, Qwen3 32B | Специализированные агенты |
| **Vision/OCR** | Qwen2.5-VL-72B (MLX) | Анализ чертежей, скан-копий |
| **Embeddings** | bge-m3 (1024 dim) через MLX | Векторный поиск (RAG) |
| **БД основная** | PostgreSQL 16 + pgvector | Документы, ВОР, БЛС, прайс-листы |
| **БД графовая** | NetworkX (in-memory + файловая сериализация) | Связи документов и норм |
| **Кэш** | Redis | Кэш состояний и результатов |
| **Поиск** | Гибридный RAG (Vector + Graph + RRF) | Семантический + контекстуальный поиск |
| **Wiki** | Obsidian (Markdown) | Правила агентов, самообучение |
| **Документы** | python-docx, PyMuPDF | Генерация DOCX, парсинг PDF |

## Ключевые компоненты

### LLMEngine с профилями
Единая точка входа для всех LLM-операций. Все модели запускаются через нативный MLX на Mac Studio:

```bash
# LLMEngine автоматически использует MLX-бэкенд
# Профиль mac_studio установлен по умолчанию
```

### БЛС — База Ловушек Строительства
Структурированная база рисков из судебной практики и Telegram-каналов. Хранится в PostgreSQL с векторными эмбеддингами (pgvector), что позволяет Юристу осуществлять семантический поиск аналогичных ситуаций через RAG.

### Map-Reduce для анализа контрактов
Длинные контракты (100+ страниц) разбиваются на чанки, каждый анализируется Юристом отдельно (map), затем результаты агрегируются (reduce). Это позволяет обрабатывать документы любого объема.

### Цикл рефлексии (самообучение)
После каждого завершенного пайплайна система анализирует аудит-логи и обновляет wiki-правила агентов. Система умнеет с каждым обработанным лотом.


### Skills Concept v2 — специализация и нормативная база

Версия v2 концепта агентных навыков вводит три ключевых изменения:

1. **Специализация компании** — система работает только с профилем компании: общестроительные, бетонные, земляные, сварочные, монтажные и шпунтовые работы. Исключённые виды (кабельные, слаботочные, вентиляция, водоснабжение, инженерные системы) помечаются как субподрядные.

2. **Актуализация нормативной базы (2026)** — обновлены ссылки на актуальные документы:
   - РД-11-02-2006 → Приказ Минстроя № 344/пр
   - РД 11-05-2007 → Приказ № 1026/пр
   - СП 543.1325800.2024 (строительный контроль)
   - ГОСТ Р 70108-2025 (электронная ИД)
   - Росстройконтроль ≠ замена ГСН

3. **MLX-only архитектура** — отказ от Ollama и профиля dev_linux. Все модели запускаются через нативный MLX-фреймворк Apple Silicon.

> См. полный документ: `docs/MAC_ASD_v11_PTO_Delo_Skills_Rework_v2.pdf`

## Быстрый старт

### 1. Инфраструктура
```bash
cd infrastructure
docker compose up -d    # PostgreSQL + Redis
```

### 2. Настройка окружения
```bash
cp .env.example .env   # Указать ASD_PROFILE, пути к БД
```

### 3. Запуск системы
```bash
source .venv/bin/activate
export PYTHONPATH=$PYTHONPATH:.
# ASD_PROFILE не нужен — MLX используется по умолчанию
python src/main.py
```

## Структура проекта

```
mac_asd/
├── src/
│   ├── agents/             # LangGraph: узлы, граф, состояние
│   │   ├── nodes.py        # Логика 7 агентов + reflection
│   │   ├── state.py        # AgentState (TypedDict)
│   │   ├── workflow.py     # StateGraph сборка
│   │   └── reflection_node.py  # Цикл самообучения
│   ├── core/
│   │   ├── llm_engine.py   # Единый LLM-интерфейс
│   │   ├── backends/       # OllamaBackend, MLXBackend
│   │   ├── rag_service.py  # Гибридный RAG (Vector+Graph)
│   │   ├── parser_engine.py # Парсинг PDF/XLSX (Vision Cascade)
│   │   ├── event_manager.py # State Machine событий проекта
│   │   ├── graph_service.py # NetworkX граф связей
│   │   └── ram_manager.py  # Управление памятью 128GB
│   ├── config.py           # Профили, модели, пути (pydantic)
│   ├── db/                 # SQLAlchemy модели, миграции
│   └── tools/              # MCP инструменты (поагентно)
├── data/
│   ├── wiki/               # Obsidian Wiki (правила агентов)
│   ├── artifacts/          # Сгенерированные документы
│   └── graphs/             # NetworkX сериализация
├── docs/                   # Проектная документация
├── agents/                 # Промпты агентов (Markdown)
└── infrastructure/         # Docker Compose
```
