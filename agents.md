## 2. Протокол "Тендерный Конвейер"

При поступлении нового лота (тендер 44-ФЗ/223-ФЗ) взаимодействие идёт по схеме:

```
Закупщик → Делопроизводитель → ПТО ─┬→ Сметчик (рентабельность + налоги)
                                     └→ Юрист   (штрафные санкции + сроки) [ПАРАЛЛЕЛЬНО]
                                              ↓
                                        Hermes → вердикт
```

### 2.1. Этап 1: Закупщик — скачивание документации

**Триггер:** Закупщик нашёл релевантный лот (регион Камчатка, шпунт/строительные работы)

**Действия:**

1. Загрузить документацию тендера из ЕИС
2. Передать Делопроизводителю на структурирование

### 2.2. Этап 2: Делопроизводитель — структурирование файлов

**Действия:**

1. Зарегистрировать все входящие документы, присвоить номера
2. Разложить по директориям:

   ```
   data/artifacts/{project_id}/
   ├── tender_docs/       # Документация тендера
   ├── drawings/          # Чертежи (от Закупщика)
   ├── vor/               # Ведомости объёмов работ
   ├── estimates/         # Сметы
   ├── legal/             # Юридические заключения
   └── registry.json
   ```

3. Уведомить Hermes: пакет готов

### 2.3. Этап 3: ПТО — проверка чертежей

**Команда от Hermes:** "Проверить ведомости на соответствие графике"

**Критическая точка:** ПТО обязан:
* Найти шпунт Л5-УМ на чертежах и подтвердить его количество
* Проверить спецификации на соответствие ГОСТ
* Выявить коллизии в проектной документации

**Выход:** Ведомость объёмов работ (ВОР) → Сметчику

### 2.4. Этап 4: Сметчик + Юрист (ПАРАЛЛЕЛЬНО)

**Сметчик:**

1. Получить ВОР от ПТО
2. Сопоставить с рыночными и сметными расценками
3. Рассчитать рентабельность и налоги
4. Выход: `{profitability, total_cost, margin_percent}`

**Юрист:**

1. Получить контракт от Hermes
2. Проверить на штрафные санкции (пени, неустойки)
3. Проверить сроки исполнения и гарантии
4. Выход: `{risk_level, penalties_found, deadline_risks}`

### 2.5. Этап 5: Hermes — вердикт

**Сборка отчёта:**
* ВОР от ПТО (объёмы подтверждены)
* Смета (рентабельность, налоги)
* Заключение юриста (риски)

**Вердикт:**
* `"Участвуем"` — риск < 20%, рентабельность > 10%
* `"Риск > 20%"` — детализация рисков

## 4. Обработка ошибок

| Ситуация                           | Действие Hermes                                        |
|------------------------------------|--------------------------------------------------------|
| Субагент вернул пустой результат   | Retry (max 2), затем эскалация на человека             |
| Субагент вернул ошибку формата     | Reject с комментарием, retry 1 раз                     |
| ПТО не нашёл шпунт на чертеже      | Пометить "шпунт не обнаруен — требует ручной проверки" |
| Смета ≠ ВОР                        | Вернуть ПТО на перепроверку                            |
| Юрист нашёл критический риск       | Включить в отчёт, поднять уровень риска                |
| Таймаут MCP (> 120s)               | Retry с увеличенным timeout, лог                       |
| Конфликт версий документов         | Делопроизводитель блокирует, уведомляет Hermes         |

## 5. Хранение состояния

* **Neo4j** — узлы проектов, документы, задачи, результаты
* **Redis** — кеш vision-results, rate-lookups, RAG-queries (TTL 24h)
* **Artifact Store** — файлы в `./data/artifacts/`
* **LightRAG** — векторные индексы: legal_rag, rates_rag
* **Registry** — `data/artifacts/{project_id}/registry.json`

Все данные RAG (LightRAG) и связи (Neo4j) хранятся в `/infrastructure/data`.

---

## 6. Технические регламенты

* **MLX** — все модели работают через MLX для прямого доступа к 128GB Unified Memory
* **Ollama** — опциональный fallback для Юриста (DeepSeek v3)
* **Heavy tasks:** Vision ПТО и логика Hermes — приоритет MLX
* **Shared models:** Qwen 3.5 32B — одна копия для Сметчик + Юрист + Закупщик

---

## 7. Правила Orchestrator'а

1. **Один тендер за раз** — Hermes ведёт один лот параллельно (Сметчик + Юрист)
2. **Логирование** — каждое действие агента = запись в граф с timestamp
3. **Идемпотентность** — повторный запуск не дублирует узлы
4. **Человек в контуре** — эскалация = уведомление + пауза до подтверждения
5. **Версионность** — все документы через Делопроизводителя
6. **Шпунт Л5-УМ** — критическая проверка. ПТО обязан найти и подтвердить
